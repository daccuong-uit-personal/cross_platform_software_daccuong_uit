# API Standards & Conventions

## 🛣️ RESTful Principles
- **Resources over Actions**: Use `/posts` (POST/GET) instead of `/createPost`.
- **Nouns only**: Use plural nouns for collections.
- **Sub-resources**: `/posts/:id/comments` for related data.
- **Versioning**: Prefix all APIs with `/v1/`.

## 📦 Response Structure (Unified Contract with FE)
*Note: This strictly follows the FE `api-contract.md` to ensure seamless communication.*

### Success Response
```json
{
  "statusCode": 200,
  "message": "Human readable message for FE toast (optional)",
  "data": {
    "id": "...",
    "..." : "..."
  },
  "meta": {
    "timestamp": "2026-05-16T12:00:00.000Z",
    "path": "/v1/resource"
  }
}
```
**Rule**: Absolutely no double wrapping (`data.data`). The core payload must be directly inside `data`.

### Error Response
```json
{
  "statusCode": 400,
  "error": "Bad Request",
  "message": "Human readable message for FE error toast",
  "path": "/v1/resource",
  "timestamp": "2026-05-16T12:00:00.000Z",
  "errors": {
    "field_name": ["Validation error message"]
  },
  "trace_id": "req-123-abc"
}
```
**Rules**:
- `statusCode` inside body MUST match the HTTP Status Code.
- `message` must be a user-friendly string, not stack traces or raw codes.
- `errors` is used for validation issues (400) mapped by field name.

## 🔢 Pagination
### Format
```json
{
  "statusCode": 200,
  "data": [
    { "id": "1", "title": "..." }
  ],
  "meta": {
    "pagination": {
      "currentPage": 1,
      "totalPages": 5,
      "totalItems": 50,
      "itemsPerPage": 10,
      "hasNext": true
    },
    "timestamp": "2026-05-16T12:00:00.000Z"
  }
}
```
- **Cursor-based**: Preferred for feeds and high-frequency data to avoid "drifting" pages (can append `nextCursor` to `meta.pagination`).
- **Offset-based**: Acceptable for static/admin tables only.

## 🛡️ Security & Auth
- **JWT Propagation**: Every request to downstream services MUST include the original user context via headers.
- **Role-Based Access Control (RBAC)**: Enforced at the service level, verified at the gateway.

## ⚡ Performance & Reliability
- **Timeouts**:
    - External/Downstream: 3s connect / 10s total.
    - Database: 5s max query time.
- **Idempotency**: All `POST` requests for resource creation SHOULD support an `X-Idempotency-Key` header.
- **Retries**: Max 3 retries with exponential backoff for 5xx errors.

## 📝 Headers
- `X-Request-ID`: Correlation ID generated by Gateway.
- `X-Trace-ID`: Distributed tracing ID.
- `X-User-ID`: Sanitized User ID from Auth Service.
